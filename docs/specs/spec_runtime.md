# Specification: Runtime Framework
Status: CURRENT (v3, 2026-04-03)

## Overview

The runtime framework (`src/runtime/`) is the agent's autonomous loop. 25+ modules, ~5800 lines of code.

Main loop: context -> plan -> audit -> execute -> commit -> sleep -> repeat.

Transport layer specifications: spec_session.md, spec_bapi.md, spec_content.md, spec_activity.md.
Data specification: spec_db.md.
Data pipeline: src/metrics/, src/memory/, src/strategy/, src/pipeline.py.

## Orchestration

### session_loop.py -- ContinuousSessionRunner
The agent's main loop with checkpoint recovery.

**Contract:**
- Loads agent_config, persona_policy, daily_plan, checkpoint
- Loop: collect context -> build directive -> generate plan -> audit -> execute -> commit
- Checkpoint is saved after plan generation; on restart -- resumes from the stop point
- Graceful shutdown via stop file or max cycles/actions
- SessionReviewer is called after each cycle

**Dependencies:** all runtime modules, SDK, DB

### main.py -- Global Orchestrator
CLI entry point. Loads settings, initializes DB, starts CycleScheduler for all accounts.

## Planning

### deterministic_planner.py -- DeterministicPlanGenerator
Generates a concrete action plan **without calling an LLM**.

**Contract:**
- Input: SessionContext, CycleDirective, PersonaPolicy, recent posts, other agent posts
- Output: AgentPlan (JSON list of actions: comment, like, follow, post)
- Comments: tiered target selection (strong/fallback) based on policy rules
- Posts: EditorialBrain (brief) -> agent writes text itself (brief_context)
- 8 variations on audit feedback (RevisionHints)

**Dependencies:** EditorialBrain, TopicReservation, CommentCoordination, NewsCooldown, PlatformLimits

### editorial_brain.py -- EditorialBrain
Selects post family and generates EditorialBrief.

**Contract:**
- Family: market_chart, news_reaction, editorial_note
- Scoring: data availability + policy adjustments + overlap penalty
- Brief: coin, angle, structure, hooks, insights, closes, visual metadata
- Template resolution: {coin}, {symbol}, {title_trimmed}, {author}, {snippet}

**Dependencies:** PersonaPolicy, ContentFingerprint

### Text Is Generated by the Agent (not code)
The planner returns `brief_context` (for posts) and `target_text` (for comments) instead of ready text.
The agent (Claude Code session) writes the text itself before execution, filling in `action.text`.
The executor checks for text presence before calling SDK.

## Audit

### plan_auditor.py -- PlanAuditor
Deterministic plan verification after generation. 9 layers:

1. **Stage rules** -- enforce post-only, reply limits, min follows, max posts
2. **Text rules** -- no trailing period, paragraph structure
3. **Comment diversity** -- similarity check (threshold 0.82) between comments in the plan
4. **Agent style** -- post length, paragraph count, coin restrictions by family
5. **Media policy** -- image attachment rules
6. **Self novelty** -- overlap with own recent posts
7. **Overlap** -- overlap with other agents' posts
8. **Reservation conflicts** -- topic lock conflicts
9. **Territory drift** -- all posts drifting outside the agent's niche

**Contract:**
- Input: AgentPlan, PersonaPolicy, recent posts, other posts, reservations
- Output: AuditResult (valid/invalid + list of AuditIssue)
- On invalid: RevisionHints are passed back to the planner

## Execution

### plan_executor.py -- PlanExecutor
Sequential plan execution via SDK.

**Contract:**
- Input: AgentPlan, SDK, start_index (for resume)
- Dispatch: comment -> sdk.comment_on_post(), like -> sdk.like_post(), follow -> sdk.follow_user(), post -> sdk.create_post(), quote_repost -> sdk.quote_repost()
- VisualPipeline for resolving images before post
- Human delay (3-15s) between actions
- Progress callback after each action

**Dependencies:** SDK, VisualPipeline

### behavior.py -- Human-like Behavior
- `warm_up()` -- page scroll before actions
- `mouse_move_to()` -- smooth mouse movement
- `delay_between_actions()` -- weighted random: 60% light, 25% medium, 10% heavy, 5% quick
- `idle_visit()` -- load a post, scroll around, leave (25% chance)

## Visuals

### visual_pipeline.py -- VisualPipeline
Resolves or generates images for posts.

**Contract:**
- Dispatch by type: image_path, chart_capture, page_capture, AI visual kinds
- AI visual kinds: market_visual, news_visual, meme_visual, personal_visual, article_cover
- Fallback: text card (PIL, 1600x800) if AI provider is unavailable
- Output: ResolvedVisual (path, kind, signature)

**Dependencies:** VisualPromptBuilder, VisualProviders, ImageNormalizer, SDK

### visual_prompt_builder.py -- Prompts for AI Visuals
Assembles LLM prompt: agent identity + action context + style guide.

### visual_providers.py -- Visual Providers
Factory: build_visual_provider() by provider_url from agent config.

### image_normalizer.py -- Normalization
PIL resize/padding to landscape format.

## Policies and Configuration

### persona_policy.py -- PersonaPolicy
Loads YAML -> frozen dataclasses.

**Key structures:**
- CoinBias -- preferred/excluded coins, bonus/penalty
- MarketAngleRules -- angle selection by price change + coin type
- StageConfig -- target actions per stage (comments, likes, posts, follows)
- StageSelectionRule -- ordered conditions (reply_limited, bootstrap, default)
- AuditStyle -- post length, paragraph count, coin restrictions
- CommentStanceConfig -- stance selection mode
- FeedScoring -- keyword bonuses/penalties
- CommentTierRule -- tiered comment targeting

### cycle_policy.py -- CycleDirective
Stage selection + sleep timing based on daily progress.

**Contract:**
- `build_cycle_directive()` -> CycleDirective (targets, stage, style/source notes)
- `choose_sleep_seconds()` -> int (next cycle)
- Overflow: if daily plan is completed -> switch to overflow stage

### agent_config.py -- ActiveAgentConfig
Runtime binding: agent_id, AdsPower profile, feed tab, action limits, session minimums, market symbols, visual config, cycle intervals, timezone.

## State Management

### daily_plan.py -- Daily Goals
Per-agent tracking: targets (like, comment, post) vs completed. Timezone-aware.

### platform_limits.py -- Platform Limits
Reply limit tracking with a 7-day window. Automatic lifting on expiry.

### post_registry.py -- Post Registry
72-hour window. Metadata: text, coin, angle, chart_symbol, visual_type, post_family, opening_signature, format_signature. Multi-agent overlap avoidance.

### topic_reservation.py -- Topic Locking
SQLite-based. Key: "{COIN}:{angle}:{md5(source)[:8]}". TTL 2 hours. reserve -> confirm/release.

### execution_checkpoint.py -- Checkpoints
Persist/load execution state for resumable cycles. JSON in data/runtime/{agent_id}/.

## Safety

### guard.py -- ActionGuard
Programmatic guardrails.

**5 layers:**
1. Global stop: 3+ action types circuit-broken -> SESSION_OVER
2. Session limit: max 80 actions (configurable)
3. Circuit breaker: 2 consecutive failures -> action type blocked
4. Daily limits: per action type (from ActionLimiter)
5. Cooldowns: post=60s, comment=30s, like=15s, follow=30s, repost=60s

**Verdict:** ALLOW / WAIT / DENIED / SESSION_OVER

### media_policy.py
Stubs for image attachment decisions. Requires implementation.

## Content Utilities

### content_fingerprint.py
Extracts semantic signals from text.

**Functions:**
- `extract_primary_coin()` -- primary ticker from text
- `infer_angle()` -- macro, rotation, ta, psychology, security, education, general
- `similarity_ratio()` -- SequenceMatcher (0-1)
- `opening_signature()` -- first 50 characters
- `format_signature()` -- normalized paragraph structure
- `add_cashtags()` -- inject $ before tickers

## Data Pipeline (separate modules)

### metrics/ (src/metrics/)
- **store.py** -- 5 SQLite tables: agent_actions, outcomes, insights, profile_snapshots, session_stats
- **collector.py** -- deferred outcome collection (6h+ after action): views, likes, comments, author_reply_rate
- **scorer.py** -- insight aggregation by dimensions (author, content_type, hour, topic, has_image) + auto-lessons

### memory/ (src/memory/)
- **compactor.py** -- generates performance.md and relationships.md from insights. Archives journal (max 5 sessions). Cleans up lessons (TTL 30 days).

### strategy/ (src/strategy/)
- **feed_filter.py** -- spam filter: keywords, token promo patterns, min chars, min likes
- **planner.py** -- bootstrap plan (deterministic) + prepare_context() for agent
- **analyst.py** -- updates strategy.md from accumulated data (every 7 sessions)
- **reviewer.py** -- session stats + journal append + lessons preparation

### pipeline.py
Orchestration: collector -> scorer -> compactor -> [analyst].
CLI: `python src/pipeline.py <agent_id> <agent_dir> [db_path]`
Each step is in try/except -- one failure doesn't block the rest.
