# Specification: Content Module

**Path:** `src/content/`
**Files:** `generator.py`, `market_data.py`, `validator.py`, `technical_analysis.py`, `news.py`

---

## Description

The Content module provides data and validation for the content pipeline. Text is generated by the persona agent (not code). The module provides:
- Market data (`get_market_data`) -- prices, volumes, 24h changes
- News (`get_crypto_news`, `get_article_content`) -- RSS + scraping
- Technical analysis (`get_ta_summary`) -- RSI, MACD, SMA/EMA
- Validation (`validate_post`, `validate_comment`) -- banned phrases, length, duplicates
- Legacy: `ContentGenerator` for generation via API (not used in v4)

The module does NOT decide what to publish or when. `ContentGenerator` and `ContentPublisher` are tools that the agent calls. The agent analyzes parser output, selects topics for each persona, and calls these functions to generate and queue content.

---

## User Stories

- As an agent, I want to generate post text in a specific persona's style with real market data, so each account's content matches its niche and contains concrete numbers.
- As an agent, I want to queue generated content for a specific account with optional scheduling, so I can create content in batches and publish later.
- As an agent, I want to get current coin prices (price, 24h change, volume), so I can include accurate data in generated content.
- As an agent, I want to get pending content from the queue, so I can publish it via `browser_actions.create_post()`.
- As an agent, I want to mark queue items as published or failed, so I can track the content lifecycle.

---

## Data Model

The module has no dedicated tables. Validation works in-memory. Market data, news, and TA are fetched from external APIs via httpx.

## API

### ContentGenerator (`src/content/generator.py`)

```python
class ContentGenerator:
    def __init__(self, provider: str, model: str, api_key: str)
```

| Method | Signature | Returns |
|--------|-----------|---------|
| `generate` | `(persona_style: str, persona_topics: list[str], topic: dict, market_data: dict) -> str` | Generated post text |

**Providers:** `"anthropic"` (Claude API) and `"openai"` (OpenAI API). The provider is selected at instantiation.

**Prompt structure:**
- System prompt: persona style, topic areas, strict writing rules (no AI cliches, conversational tone, $CASHTAGS, etc.)
- User prompt: topic name + trending hashtags + related coins + current market data (prices, 24h change)

The system prompt contains rules from `config/content_rules.yaml` inline (not loaded from file -- rules are embedded in the prompt string).

### ContentPublisher (`src/content/publisher.py`)

```python
class ContentPublisher:
    def __init__(self, client: BapiClient, db_path: str)
```

| Method | Signature | Returns |
|--------|-----------|---------|
| `publish` | `(text: str, hashtags: list[str] \| None = None) -> dict` | Calls `BapiClient.create_post()` -- currently raises `NotImplementedError` |
| `queue_content` | `(account_id: str, text: str, hashtags: list[str] \| None, topic: str, meta: dict \| None, scheduled_at: str \| None) -> int` | Queue row ID |
| `get_pending` | `(account_id: str) -> list[dict]` | Pending items where `scheduled_at <= now()` |
| `mark_published` | `(queue_id: int, post_id: str = "") -> None` | Sets status='published' |
| `mark_failed` | `(queue_id: int, error: str) -> None` | Sets status='failed' |

**Note:** `publish()` is a stub. Actual publishing goes through `session.browser_actions.create_post()` because Binance requires a client-side nonce + signature. The publisher manages only the queue.

---

### get_market_data (`src/content/market_data.py`)

```python
async def get_market_data(symbols: list[str]) -> dict[str, dict[str, Any]]
```

Fetches 24h ticker data from the public Binance API (`https://api.binance.com/api/v3/ticker/24hr`) for each symbol (e.g., `["BTC", "ETH"]`). Appends `USDT` to each symbol for the API call.

**Returns:**
```python
{
    "BTC": {"price": 67500.0, "change_24h": 2.3, "volume": 15000.0},
    "ETH": {"price": 3800.0, "change_24h": -1.2, "volume": 8000.0},
}
```

Fetches all symbols in parallel via `asyncio.gather()`. Failed fetches are logged and excluded from results.

## Business Logic

### Current Pipeline (v3, runtime)
```
SessionContextBuilder collects context (market, news, TA, feed)
  -> DeterministicPlanGenerator generates a plan (brief_context, no text)
     -> EditorialBrain selects family + brief (coin, angle, hooks)
  -> PlanAuditor validates (style, overlap, coordination)
  -> Agent (Claude Code session) writes post and comment text
  -> PlanExecutor executes via SDK
     -> sdk.create_post(text, coin, sentiment, image_path)
     -> browser_actions.create_post(page=persistent_page, ...)
```

In this pipeline, `ContentGenerator` and `ContentPublisher` **are not used**. Text is generated by the agent (Claude Code session) itself, using brief_context from the plan and its own files (style.md, strategy.md). Market data (`get_market_data`, `get_crypto_news`, `get_ta_summary`) is used by `SessionContextBuilder` to build context.

### Legacy Pipeline (scheduler)
> Only used in `src/scheduler/scheduler.py`. Not active in the current deployment.

```
Agent decides to create a post for account X
  -> ContentGenerator.generate(style, topics, topic, market_data)
  -> ContentPublisher.queue_content(account_id, text, ...)
  -> ContentPublisher.get_pending(account_id)
  -> browser_actions.create_post(ws, text, coin, sentiment)
  -> ContentPublisher.mark_published(queue_id, post_id)
```

### Prompt Construction
System prompt includes:
- Persona style and topics
- Writing rules (conversational style, no AI cliches, $CASHTAGS, concrete numbers)
- Banned phrases list (17 phrases)
- Good and bad examples
- Length recommendations: 100-280 characters for short notes, up to 1000 for analysis

User prompt includes:
- Topic name
- Up to 5 trending hashtags (with # prefix)
- Up to 5 related coins (with $ prefix)
- Current market data in format `$COIN: $price (+/-change% 24h)`

### Queue Scheduling
Items with `scheduled_at = NULL` are immediately available for publishing. Items with a future `scheduled_at` become available after the time arrives. `get_pending()` checks `scheduled_at <= utcnow()`.

## Edge Cases

| Scenario | Expected Behavior |
|----------|-------------------|
| AI API returns an empty response | `generate()` returns an empty string -- agent should check and retry or skip |
| AI API rate limited or unavailable | Exception propagated to caller (agent manages retry logic) |
| `market_data` is an empty dict | User prompt skips the market section -- AI generates without specific prices |
| Symbol not found on Binance | `_fetch_ticker` raises an exception, logged as warning, symbol excluded from results |
| No pending items in the queue | `get_pending()` returns an empty list |
| `publish()` called | Raises `NotImplementedError` -- use `browser_actions.create_post()` |
| Duplicate content in queue | No deduplication -- the same text can be queued multiple times |

---

## Priority and Dependencies

- **Priority:** High
- **Depends on:** `src/bapi/client.py` (market_data, news use httpx)
- **Blocks:** Publishing pipeline in the scheduler (content must be generated before it can be published)

## Additional Modules (added in v3)

### news.py (151 lines) -- Crypto News from RSS

```python
async def get_crypto_news(limit: int = 10) -> list[dict]
async def get_article_content(url: str) -> dict
```

RSS sources: CoinDesk, CoinTelegraph, Decrypt. Manual XML parsing (regex, no XML libraries).

**Returns:** `[{"title": str, "source": str, "url": str, "published_at": str, "summary": str}]`

`get_article_content()` scrapes full article text by URL. Filters noise: price tables, SVG/CSS, navigation.

### technical_analysis.py (214 lines) -- Technical Analysis

```python
async def get_ta_summary(symbol: str, timeframe: str = "1d") -> dict
```

Pure Python implementation without external TA libraries:
- SMA, EMA (arbitrary period)
- RSI (14 periods)
- MACD + signal line + cross detection
- Support / Resistance (from recent highs/lows)

Data: 200 candles from Binance API (`/api/v3/klines`). Minimum 50 candles required for calculation.

### validator.py (392 lines) -- Content Validation

Validation rules before publishing:

| Function | What It Checks |
|----------|----------------|
| `validate_post(text, recent_posts)` | banned phrases, min 2 paragraphs, min 80 chars, duplicates (threshold 0.65), topic repeat |
| `validate_comment(text)` | 5-500 chars, no generic comments, banned phrases |
| `validate_article(title, body)` | title 10-200 chars, body 300+ chars, 3+ paragraphs |
| `validate_quote(comment)` | same as post: 2 paragraphs, 80+ chars |
| `verify_prices(text, market_data)` | prices in text within +/-10% of actual |

Banned phrases are loaded from `config/content_rules.yaml` with hardcoded fallback.
Integrated into SDK: `create_post`, `create_article`, `quote_repost`, `comment_on_post`.
